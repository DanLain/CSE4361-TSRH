Test Help!

The Test page helps you create and find the test details of a patient. It has both the test request and the test results fucntionality under it. For a Patient a test request must be first created only after which you can associate a test result with it. You can begin with searching a patient for whom test requests are to be created.

  • Patient Search

    Patient search has three fields in which you can enter the input for any or all or combination of the three fields to search for a patient record.

    • Enter a patient number.
    • And /Or enter the patient name - remember to enter it as (lastname,firstname).
    • And /Or enter Date of birth, select a date using the date picker, you cannot enter. In the date picker you can click on the month-year displayed between the left and right arrows for more options to select, to enter.
    • In case you make any mistakes you can reset the fields by using the 'reset' button.
    • you can now click on the 'submit' button and retrieve your patient record.

  • Patient Details

    This page displays the patient details of the patient you searched for.

    • You can click on 'Test Request' and find out if the patient has any test requests.
    • You can click 'Back' button to return back and search for a new patient.

  • Test Requests

    • This page gives all the patient test requests that exists for the particular patient, in a table. You can choose any of the records and perform more operations on it like - viewing more information about the test request, viewing the associated test result, creating a new test result or alerting for the test request.

    • If a test request exists for the patient: a table is displayed, else the page has just two buttons i.e. 'New test request' button to create a new test request for the patient and a 'Back' button to go back to the patient details page.
    • In the table associated with each test request is a radio button which upon choosing you may perform the operations listed.
    • There is a search option above the table. By entering the search keyword you could filter the list of records being displayed to match the entered criteria
    • 'View' button displays more information about the test request and is readonly. You cannot modify its content.
    • 'View Test Result' button can be used to check the details of the test result, if there are any associated with the Test request.
    • 'New Test Result' button can be used to create a test result, associated with the Test request.
    • The 'Alert' button lets you set an alert for the test request and intimate the required people.
    • You also have two buttons at the bottom of the page which are not associated with any test requests i.e. 'New test request' button and a 'Back' button.

  • Test Results

    • This page gives all the patient test results that exists for the particular patient, in a table. You can choose any of the records and perform more operations on it like - viewing more information about the test result and alerting for the test result.

    • If a test result exists for the patient: a table is displayed, else the page has just one button i.e. a 'Back' button to go back to the patient details page.
    • In the table associated with each test result is a radio button which upon choosing you may perform the operations listed.
    • There is a search option above the table. By entering the search keyword you could filter the list of records being displayed to match the entered criteria
    • 'View' button displays more information about the test result and is readonly. You cannot modify its content.
    • The 'Alert' button lets you set an alert for the test result and intimate the required people.

  • Create New Test Request

    • This page upon successfully completing adds a new Test Request to the patient

    • You have a list of fields and its corresponding values for the patient test request that are to be filled up.
    • Any of the date fields can only be picked from the date picker and cannot be manually entered.
    • All filed names preceeding with a '*' mark are required to be filled, others can be left blank.
    • The 'User Name' field gets auto filled and the user cannot edit this field.
    • In the end of the page you have an 'Save' button which when clicked takes you to the earlier Test Requests page.

  • Create New Test Result

    • This page upon successfully completing adds a new Test Result to the patient

    • You have a list of fields and its corresponding values for the patient test result that are to be filled up.
    • Any of the date fields can only be picked from the date picker and cannot be manually entered.
    • All filed names preceeding with a '*' mark are required to be filled, others can be left blank.
    • The 'User Name' and 'Test Result Number' fields gets auto filled and the user cannot edit these fields.
    • In the end of the page you have an 'Save' button which when clicked takes you to the earlier Test Requests page.

  • Information Of Test Request

    • This is just an informational page on the patient's test request and no fields can be modified here.

    • You have a list of fields and its corresponding values for the patient test request.
    • In the end of the page you have an 'Ok' button which when clicked takes you to the earlier Test Requests page.

  • Information Of Test Result

    • This is just an informational page on the patient's test result and no fields can be modified here.

    • You have a list of fields and its corresponding values for the patient test result.
    • In the end of the page you have an 'Ok' button which when clicked takes you to the earlier Test Requests page.
